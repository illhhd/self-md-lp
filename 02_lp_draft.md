# self.md LP構成案

## LP全体の設計思想

**ターゲット：** シリーズA〜Bのスタートアップ創業者
**CVポイント：** 無料の「ギャップ診断セッション（30分）」への申し込み
**トーン：** 知的で静か。煽らない。「わかってる人が書いている」感。
**長さ：** スクロール7〜8画面分

---

## セクション構成

---

### Section 0: ファーストビュー

```
考えはある。
でも、伝わっていない。
```

サブコピー：
「経営者の頭の中を整理し、会議・採用・提案で"そのまま使える形"に変えるサービス」

CTA：まず、30分の無料診断を受ける →

背景：余白多め、テキスト中心。写真不要。
色：白背景 × 黒テキスト × アクセントに深いブルーか深緑。

---

### Section 1: 問題提起 —「伝わっていない」の正体

**見出し：**
```
あなたは説明している。
でも、届いていない。
```

**本文（箇条書き形式）：**

経営者はいつも、いろんな相手に同じことを説明しています。

- 社員には、理念として
- 採用候補者には、魅力として
- 取引先には、方針として
- 投資家には、勝ち筋として

でも、こんな経験はありませんか。

- 「言ったはずなのに、伝わっていなかった」
- 「毎回同じことをゼロから説明するのが重い」
- 「自分では論理的に話しているつもりなのに、感覚的だと言われる」
- 「任せたつもりなのに、急にひっくり返したと言われる」

**問題は、考えがないことではありません。
考えはあるのに、自分が思っている伝わり方と、実際の伝わり方がズレていること。**

そのズレが、会議を長くし、採用を難しくし、組織にノイズを生んでいます。

---

### Section 2: なぜズレるのか — ギャップの構造

**見出し：**
```
経営者が見ている自分と、
周囲が見ている自分は、違う。
```

**本文：**

これは能力の問題ではありません。構造の問題です。

| あなたの自己認識 | 周囲の認識 |
|---|---|
| 合理的に判断している | 直感で決めている |
| ちゃんと説明している | 結論だけ言われる |
| 任せている | 急にひっくり返される |
| オープンにしている | 本音が見えない |

**このギャップに気づかないまま、いくら自分の考えを"整理"しても、
出てくるのは"理想の自己像"の言語化です。**

self.md は、このギャップを発見するところから始めます。

---

### Section 3: self.md とは何か

**見出し：**
```
経営者の"自己認識"と"周囲の認識"を突き合わせ、
仕事で使える"説明の基盤"をつくる。
```

**本文：**

self.md は、3つのステップで動きます。

**Step 1. 引き出す（人間が行う）**
経営者本人への深掘りインタビュー。
加えて、共同創業者・幹部・社員への周囲インタビュー。
あなたの頭の中と、周囲の頭の中を、両方聞きます。

**Step 2. ズレを見つける（人間 + AI）**
本人の自己認識と周囲の認識を突き合わせ、
ギャップを構造的に可視化します。
ここが、コーチングでもAIツールでもできない部分です。

**Step 3. 使える形にする（AI + 人間レビュー）**
発見したギャップを踏まえて、
会議・採用・提案・社内共有でそのまま使える出力を生成します。

**ビジュアル案：**
3ステップを横並びのフロー図で。
Step 1とStep 2に「人間のアイコン」、Step 3に「AIのアイコン」を添えて、
「核は人間がやる。AIは出力の道具」を視覚的に示す。

---

### Section 4: 4つの出力 — 何が手に入るのか

**見出し：**
```
整理して終わりではなく、
仕事でそのまま使える形にする。
```

**カード形式で4つ並べる：**

---

**Founder Brief**
経営者の"取扱説明書"

あなたの判断軸、思考の癖、強み、地雷。
本人の自己認識と周囲の認識を突き合わせた"実像"を一枚に。
自分で見返すためにも、周囲と共有するためにも使えます。

「この人にはこう話すと通る」がわかる資料。

---

**Meeting Brief**
会議前の"地雷マップ"

今回の会議で何がすれ違いそうか。
どこで「言ったつもり」が発生しそうか。
先に合意すべき前提は何か。
会議の前に5分で読めるメモとして提供。

---

**Stakeholder Narrative**
相手別の"翻訳シート"

あなたの考えを、採用候補者・社員・取引先・投資家に
それぞれ伝わる言葉に変換。
「言いがちだけど伝わらない表現」と
「こう言い換えると届く表現」の対照表つき。

---

**Project Premise Generator**
企画の"前提設計シート"

企画や提案が通るかどうかは、情報量より問題設定で決まる。
あなたの思考パターンに合った問題設定と価値訴求を、
周囲がどう受け取るかも含めて設計。

---

### Section 5: 「それ、AIでできるのでは？」への回答

**見出し：**
```
ChatGPTに「自分の考えを整理して」と頼んだこと、
ありませんか？
```

**本文：**

AIは、あなたが入力した情報を整理するのは得意です。
でも、あなたが入力しなかった情報は、整理できません。

- あなたが気づいていない判断の癖
- 周囲があなたに対して感じているけど言えていないこと
- あなたが「伝わっている」と思い込んでいるズレ

これらは、自分ひとりでは見えません。
AIに相談しても、出てくるのは「あなたが思っている自分」の整理版です。

**self.md の核は、AIではなく「引き出し」にあります。**

人間が対話で引き出し、周囲の声と突き合わせ、
そこで初めて見えた構造をAIで出力する。

AIは道具です。self.md の価値は、その手前にあります。

---

### Section 6: 誰のためのサービスか

**見出し：**
```
こんな経営者のためにつくりました。
```

**チェックリスト形式：**

- [ ] 組織が10〜50人になり、全員に直接話す限界を感じている
- [ ] 採用面接で毎回同じ話をゼロからするのが重い
- [ ] 投資家への説明と社員への説明を、毎回別々に考えている
- [ ] 「任せたい」のに、任せた結果にモヤモヤする
- [ ] 自分の判断に一貫性がある自信はあるが、周囲にはそう見えていない気がする
- [ ] 会議で「それ前にも言ったよね」と思うことが多い

**2つ以上当てはまるなら、self.md が役に立つかもしれません。**

---

### Section 7: サービスの流れ

**見出し：**
```
最短2週間で、あなたの"説明の基盤"ができます。
```

**タイムライン形式：**

```
Week 1
├── Day 1-2: 経営者本人インタビュー（90分 × 2回）
├── Day 3-5: 周囲インタビュー（3〜5名、各30分）
└── Day 5: 過去の意思決定事例の収集

Week 2
├── Day 6-8: 思考構造の整理 + ギャップ分析
├── Day 9-10: self.md（Founder Brief）作成
├── Day 10: 4つの出力の生成
└── Day 11: フィードバックセッション（ギャップの共有と対話）

Week 3〜
└── 修正・確定 → 運用開始
```

---

### Section 8: 継続の仕組み

**見出し：**
```
一度つくって終わりではない。
経営者の考えは変わるから。
```

**本文：**

self.md は、3つのタイミングで更新します。

1. **四半期更新** — 定期的な軽量チェック。考えの変化を反映。
2. **イベント更新** — 資金調達前、大型採用前、新規事業立ち上げ前に集中的に更新。
3. **効果トラッキング** — 採用通過率、会議時間、社員理解度スコアを定期測定。

**あなたの"説明の基盤"は、会社の成長に合わせてアップデートされ続けます。**

**見出し：**
```
まず、30分であなたの"ギャップ"を覗いてみませんか。
```

無料の「ギャップ診断セッション」では、
30分の対話を通じて、あなたの自己認識に

営業トークはしません。
「自分にズレがあるのか、ないのか」を知るだけでも価値があるはずです。

**CTA：** 無料ギャップ診断を予約する →

## LP デザインノート

### トーン＆マナー
- **余白多め、テキスト主体。** 写真やイラストは最小限。
- **色：** 白背景 × 黒テキスト × ダークブルーまたはダークグリーンのアクセント。
- **フォント：** 和文はNoto Sans JPまたは游ゴシック。欧文はInter。
- **煽らない。** 「今すぐ」「限定」「残りわずか」は使わない。
- 知的で静か。読んでいて頭が良くなる感じ。

### 想定ターゲットのペルソナ
- 30〜45歳のスタートアップ創業者
- シリーズA〜B、社員10〜50人
- テック系だが、本人は必ずしもエンジニアではない
- 自分の考えには自信があるが、伝え方に課題を感じている
- コーチングは「ちょっと違う」と思っている層

### CTA設計
- メインCTA：無料ギャップ診断セッション（30分、オンライン）
- サブCTA：サービス資料ダウンロード（メールアドレス取得）
- CTAボタンは Section 0（ファーストビュー）と Section 9（最後）の2箇所に配置
- 途中セクションにはCTAを入れない（押し売り感を出さない）

### モバイル対応
- セクション間の余白を十分に取る
- テーブルはモバイルではカード形式に変換
- フォントサイズは本文16px以上
